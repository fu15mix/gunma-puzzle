# 群馬県市町村パズル Webアプリ MVP 開発指示書

## 1. 開発目的

React + TypeScript + Viteで「群馬県市町村パズル」のWebアプリMVPを作成してください。

Windows PCで開発し、iPhone/iPadのSafariでも遊べるWebアプリにしたいです。

最初は本物の群馬県地図データではなく、仮のSVGピースで動作確認できるMVPを作ってください。  
後から群馬県35市町村のSVG pathデータに差し替えられる構造にしてください。

---

## 2. アプリ概要

群馬県の市町村ピースをドラッグして、正しい場所の近くに置いたら自動で吸着する地図パズルです。

ユーザーは市町村ピースをドラッグし、パズル盤上の正しい位置に配置します。  
正解位置の近くで離すと、ピースが自動で正しい位置に吸着します。  
すべてのピースを配置できたら「クリア！」と表示します。

---

## 3. MVP仕様

以下の機能を実装してください。

- SVGピースをドラッグできる
- マウス操作とタッチ操作の両方に対応する
- 正解位置の近くに置いたら自動で吸着する
- 吸着したピースは固定される
- 間違った場所に置いた場合は元の位置に戻る
- すべてのピースを置いたら「クリア！」と表示する
- リセットボタンで最初からやり直せる
- iPhone/iPadのSafariでも操作しやすいレイアウトにする

## 6. データ構造

`src/data/pieces.ts` を作成し、ピースデータを分離してください。

座標の基準は、各ピースの `path` を描画する基準座標とします。  
`currentX/currentY` と `correctX/correctY` は、いずれもこの基準座標で扱ってください。

`Piece` 型は以下のようにしてください。

```ts
export type Piece = {
  id: string;
  name: string;
  path: string;
  labelX: number;
  labelY: number;
  correctX: number;
  correctY: number;
  startX: number;
  startY: number;
  currentX: number;
  currentY: number;
  placed: boolean;
};
```

仮データ例:

```ts
export const initialPieces: Piece[] = [
  {
    id: "maebashi",
    name: "前橋市",
    path: "M0 0 L80 0 L80 50 L0 50 Z",
    labelX: 40,
    labelY: 28,
    correctX: 220,
    correctY: 180,
    startX: 20,
    startY: 520,
    currentX: 20,
    currentY: 520,
    placed: false,
  },
];
```

## 9. PuzzleBoard.tsxの役割

`PuzzleBoard.tsx` では以下を担当してください。

- SVGのパズル盤を表示する
- 地図のガイド枠を表示する
- ピースを表示する
- Pointer Eventsでドラッグ処理を行う
- ドラッグ開始時に対象ピースを最前面に移動する
- ドラッグ終了時に吸着判定を行う
- 正解したピースを固定する
- 不正解の場合はピースを元の位置へ戻す

---

## 10. 吸着判定

ドラッグ終了時に、ピースの現在座標と正解座標の距離を計算してください。

```ts
const distance = Math.sqrt(
  (piece.currentX - piece.correctX) ** 2 +
  (piece.currentY - piece.correctY) ** 2
);
```

距離が一定以下なら正解扱いにしてください。

最初は一律で以下の値を使ってください。

```ts
const SNAP_DISTANCE = 50;
```

正解した場合:

- `currentX` を `correctX` にする
- `currentY` を `correctY` にする
- `placed` を `true` にする
- ピースを固定して再ドラッグできないようにする

不正解の場合:

- `currentX` を `startX` に戻す
- `currentY` を `startY` に戻す

+--------------------------------------+

## 12. ピース表示

各ピースはSVGの `<path>` で表示してください。

ピースには市町村名を表示してください。

ラベル位置はピースごとに `labelX` と `labelY` を持たせて制御してください。  
市町村名は各ピースの中央付近に読みやすく配置し、将来の実データ差し替え時にも位置調整しやすい構造にしてください。

- ピース本体: `<path>`
- 市町村名: `<text>`

配置済みピースは、未配置ピースと見た目を少し変えてください。

例:

- 未配置: 明るい色
- 配置済み: 少し濃い色
- ガイド: 薄いグレー

色は見やすければ仮で構いません。

また、ドラッグ中のピースは操作対象が分かりやすいように、影・拡大・透明度変更などで見た目を少し変えてください。

## 13. スマホ・タブレット対応

CSSでレスポンシブ対応してください。

要件:

- 画面幅が小さい場合もSVGが画面内に収まる
- iPhone/iPadのSafariで操作できる
- タッチ操作しやすいようにボタンと文字を大きめにする
- パズル部分には `touch-action: none;` を設定する
- ピース名は読みやすく表示する
- スクロール誤操作が起きにくいようにする

確認対象の目安は以下とします。

- iPhoneの縦持ち幅
- iPadの縦持ち・横持ち

上記でレイアウト崩れや操作不能が起きないことを目標にしてください。

---

## 14. Pointer Eventsの方針

マウスとタッチを共通で扱うため、Pointer Eventsを使ってください。

使用するイベント:

- `onPointerDown`
- `onPointerMove`
- `onPointerUp`
- `onPointerCancel`

ドラッグ中は、対象ピースが指またはマウスに追従するようにしてください。

SVG座標と画面座標の変換が必要な場合は、SVGの `getScreenCTM()` を使って座標変換してください。

---

## 15. リセット仕様

リセットボタンを押したら、全ピースを初期状態に戻してください。

リセット後:

- `currentX = startX`
- `currentY = startY`
- `placed = false`
- 正解数は0に戻る
- クリア表示は消える

---

## 16. クリア判定

すべてのピースの `placed` が `true` になったらクリアとします。

```ts
const isCleared = pieces.every((piece) => piece.placed);
```

クリア時は、画面上に以下を表示してください。

```txt
クリア！
```

クリア後もリセットボタンは利用可能とし、表示はそのまま維持して構いません。
